# Media Archive

The Media Archive is Spora's indexed, filterable library of media
generated by plugins (images, audio, video, documents). Without it, an
operator asking "show me all the images agent X generated last week"
would have to scroll chat history and parse markdown URLs by hand —
because nothing currently tracks the bytes the chat referenced. The
archive fixes that: every media-producing tool call leaves a row in
`media_assets` that's addressable by agent, plugin, tool, type, and
time, plus the operator can browse and download the originals.

The archive is **explicitly opt-in per tool call** — there is no
auto-archive hook because the core can't safely guess whether a tool's
output is bytes, hex, base64, or a CDN URL. Plugin authors call
`MediaArchiveService::ingest()` with the form they have in hand; the
core does the rest (fetch, sniff, persist, index).

---

## The contract

A single method, `MediaArchiveService::ingest(MediaIngestRequest)`, with
four input forms selected by which DTO field is set:

| Input form | DTO field | Plugin code shape | Core skips | Core still runs |
|---|---|---|---|---|
| URL (default, primary) | `url` | `ingest(url: $cdnUrl, …)` | — | fetch + sniff + image-meta + ffprobe |
| Raw bytes | `bytes` | `ingest(bytes: $bin, …)` | fetch | sniff + image-meta + ffprobe |
| Hex bytes | `hex` | `ingest(hex: $hexString, …)` | fetch | `hex2bin()` + sniff + image-meta + ffprobe |
| Base64 | `base64` | `ingest(base64: $b64, …)` | fetch | `base64_decode(strict)` + sniff + image-meta + ffprobe |

The DTO rejects requests with zero or more than one input form set. The
remaining fields are optional context the core can't infer:

```php
final readonly class MediaIngestRequest
{
    public function __construct(
        public ?string $bytes = null,
        public ?string $hex = null,
        public ?string $base64 = null,
        public ?string $url = null,
        public ?string $mime = null,
        public ?string $filename = null,
        public ?MediaType $mediaType = null,
        public ?int $agentId = null,
        public ?int $taskId = null,
        public ?int $toolCallId = null,
        public ?string $pluginSlug = null,
        public ?string $toolName = null,
        public ?string $prompt = null,
        public ?int $width = null,
        public ?int $height = null,
        public ?float $durationSeconds = null,
        public ?int $byteSize = null,
        /** @var array<string>|null */
        public ?array $tags = null,
        /** @var array<string, mixed>|null */
        public ?array $metadata = null,
    ) {}
}
```

### `MediaType`

```php
enum MediaType: string
{
    case Image = 'image';
    case Audio = 'audio';
    case Video = 'video';
    case Document = 'document';
    case Unknown = 'unknown';
}
```

Set `mediaType` explicitly when the tool knows the format (a tool that
only ever produces PNG can skip the sniff for type), otherwise the
service derives it from the sniffed MIME.

## What the core does, in order

For the **URL branch**:

1. **HEAD probe** the URL — read `Content-Type` and `Content-Length`.
2. If `media_archive.max_promote_bytes` would be exceeded, or
   `media_archive.promote_external = false`, skip the body fetch and
   store the row as `external` (the URL becomes `asset_url`; no bytes
   are fetched).
3. **GET** the body with a configurable timeout and follow-redirects.
   Stream to `AssetStore::store()` — never holds the whole file in
   memory for large media.
4. **MIME sniff** from the bytes (`finfo` + magic-byte table). The
   sniff wins over the caller's hint.
5. **Image metadata** via `getimagesize()` — populates `width`,
   `height`, and corrects `mime_type` if `getimagesize` disagrees with
   `finfo`.
6. **Audio/video metadata** is best-effort: shell out to `ffprobe` when
   `media_archive.ffprobe_enabled = true` AND `ffprobe` is on PATH.
   When disabled or missing, `duration_seconds` stays `null`. The
   operator can backfill via a future admin-UI edit.
7. **Persist** the row. Idempotent on `(tool_call_id, asset_url)`.

For `bytes` / `hex` / `base64`, the URL branch is skipped — bytes are
processed the same way from step 4 onwards.

### Failure handling

- `hex` / `base64` decode failure → `InvalidArgumentException`.
- URL fetch non-2xx, oversized body, transport failure → the service
  falls back to `storage_mode = external` with the original URL
  preserved, so the row still has *some* record even when the CDN
  goes away. Operators can retry ingest later if needed.

## Promote vs external

The archive always indexes metadata; whether it also keeps a local copy
of the bytes is a per-row policy decision controlled by
`media_archive.promote_external`:

| Setting | Behaviour |
|---|---|
| `promote_external = true` (default) | URL inputs are fetched and stored locally via `AssetStore`. `storage_mode = local` (or `data_url` for small payloads in `auto` mode). `source_url` preserves the original CDN URL. |
| `promote_external = false` | URL inputs are stored as `storage_mode = external` without fetching the body. The CDN URL becomes `asset_url` directly. |

A `media_archive.max_promote_bytes` guard (default 100 MiB) prevents
accidental promotion of multi-GB videos — when the HEAD probe reports
`Content-Length` above the threshold the row is recorded as `external`
regardless of `promote_external`. Useful when a CDN serves a 50 MB
preview at one URL and a 4 GB master at another; the archive captures
the metadata without filling disk.

## Plugin opt-in (the easy path)

Existing plugin tools using the `StoresBinaryAssets` trait get a
parallel `archiveMedia(...)` helper that wraps the typical "I have raw
bytes, write them, sniff, record" flow into one call:

```php
final class MiniMaxImageTool extends AbstractTool {
    use StoresBinaryAssets;

    protected function doWork(ToolContext $ctx, array $args): ToolResult {
        // ... existing fetch + embedHex() flow ...
        [$url, $mode] = $this->embedHex($hex, mime: 'image/png', filename: 'minimax.png');

        // One-line opt-in to the archive:
        $this->archiveMedia(
            bytes: $decoded,
            mime: 'image/png',
            filename: 'minimax.png',
            mediaType: MediaType::Image,
            context: [
                'agentId'    => $ctx->agentId,
                'taskId'     => $ctx->taskId,
                'toolCallId' => $ctx->toolCallRecord->id,
                'pluginSlug' => 'minimax',
                'toolName'   => 'image',
                'prompt'     => $args['prompt'],
            ],
        );

        return new ToolResult(true, $content, ['asset_mode' => $mode]);
    }
}
```

For URL output (CDN-hosted media), call the service directly:

```php
final class MiniMaxVideoTool extends AbstractTool {
    public function __construct(
        private readonly MediaArchiveService $mediaArchive,
        // ...
    ) {}

    protected function doWork(ToolContext $ctx, array $args): ToolResult {
        // ... existing fetch ...
        $cdnUrl = $response['video_url'];

        $this->mediaArchive->ingest(new MediaIngestRequest(
            url: $cdnUrl,
            agentId: $ctx->agentId,
            taskId: $ctx->taskId,
            toolCallId: $ctx->toolCallRecord->id,
            pluginSlug: 'minimax',
            toolName: 'video',
            prompt: $args['prompt'],
        ));

        return new ToolResult(true, MediaEmbed::videoFromUrl($cdnUrl));
    }
}
```

## Retention

The archive doesn't enforce a retention window — operators own the
`media_assets` table. The two `bin/spora media:*` commands handle the
common cleanup flows:

```bash
# Browse everything an agent has produced
bin/spora media:list --agent=42 --type=image --since="2026-06-01" --json | jq

# Sweep rows whose on-disk asset is gone (storage_mode = local + file missing)
bin/spora media:gc --max-age-days=30 --dry-run
bin/spora media:gc --max-age-days=30

# Drop the orphaned bytes themselves (separate pass — AssetStore's GC)
bin/spora assets:gc --max-age-days=7
```

`media:gc` only touches rows whose `storage_mode = local` AND whose
on-disk file is missing — external rows are allowed to outlive their
CDN because the row IS the durable record.

## Configuration

| Env var | Default | Effect |
|---|---|---|
| `SPORA_MEDIA_ARCHIVE_PROMOTE_EXTERNAL` | `true` | When `false`, URL inputs are stored as `external` without fetching |
| `SPORA_MEDIA_ARCHIVE_FETCH_TIMEOUT` | `30` (seconds) | Ceiling for the HEAD + GET path |
| `SPORA_MEDIA_ARCHIVE_MAX_PROMOTE_BYTES` | `104857600` (100 MiB) | CDN responses larger than this are recorded as `external` regardless of `promote_external` |
| `SPORA_MEDIA_ARCHIVE_FFPROBE_ENABLED` | `false` | When `true` (and `ffprobe` is on PATH), audio/video duration is extracted |

Settings are merged from defaults → `config.php` → env vars, same as
every other `SPORA_*` setting.

## What the archive does NOT do

- **No auto-archive.** Plugins opt in via `ingest()`. The core doesn't
  parse chat markdown or scan for media references — that would silently
  archive every URL the LLM happened to mention, which is wrong.
- **No de-duplication across tool calls.** Each `ingest()` is a single
  row, even if the same bytes were archived by a previous call.
  `(tool_call_id, asset_url)` is the unique index, so the same tool call
  pointing at the same URL is idempotent — different tool calls are not.
- **No bytes deletion on `DELETE /api/v1/media/{id}`.** The endpoint
  removes the row, but the on-disk asset stays until `bin/spora
  assets:gc` reaps it.
